# 貢獻指南

很開心你想貢獻 🙌 這裡很輕鬆,幾點小提醒就好:

## 流程

1. Fork → 開一個分支 → 送 PR。
2. **一個 PR 專注一件事**(新增一個 skill,或改一個 skill),最好 review。
3. PR 描述講一下:這個 skill 做什麼、什麼情境會觸發。

## 🔒 安全(重要,但別緊張)

這些 skill 會裝進別人的電腦,所以安全優先 🙏

- **純 markdown 的 skill 我會盡快看。**
- 如果你的貢獻**包含腳本(`.sh` / `.py` 等)或會自動執行的 hooks / `settings.json`**,我會**逐行確認再合併**,可能多花點時間,還請見諒——在 PR 裡說明「它為什麼需要、做了什麼」能加速審查。
- 這不是不信任你,是對每一位安裝者負責 🙂

## 風格

- `SKILL.md` 要有清楚的 `name`,以及描述「**什麼時候該用**」的 `description`(觸發才會準)。
- 附上簡短的用法/觸發範例。
- 中文正文用全形標點、程式碼區塊用半形(這是我的習慣,PR 我可能會順手調一下)。

就這樣,期待你的 skill!有任何問題,開個 issue 聊聊都可以。
