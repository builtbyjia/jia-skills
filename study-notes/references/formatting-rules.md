# 格式慣例(排版與標點)

這些規則對筆記的閱讀體驗影響很大,務必套用。

## 標點:全形 vs 半形

| 情境 | 符號 |
|---|---|
| 中文正文的冒號、分號、逗號、問號 | 全形 `：` `；` `，` `？` |
| 程式碼(``` 區塊、行內 `code`、路由 `:id`、語法本身) | 半形 `:` `;` `,` `?` |
| 英文語句/片語中的符號 | 依情境用半形(看具體情況) |

判斷訣竅:**符號緊鄰中文字** → 用全形;**符號屬於程式碼或英文語境** → 用半形。

範例:
- ✅ `這份作業：實作會員 CRUD，並用 Jest 驗收。`(正文全形)
- ✅ `` `Number(req.params.id)` `` (行內程式碼半形)
- ✅ ```js\nconst x = { id: 1, name: 'a' };\n```(程式碼區塊半形)

## markdown 引用區塊(`>`)斷行

引用區塊裡若要讓兩行**各自獨立成行**,中間必須放一個「只有 `>` 的空行」,否則 markdown 會把連續的行合併成同一段。

```markdown
> ⚠️ 第一個提示。
>
> ⚠️ 第二個提示。
```

## 檔案之間的連結

四個檔案互相用相對連結串起來,方便跳轉:
- 每份筆記頂部放 `[← 回索引](./README.md)`
- 每份筆記底部放 `**下一份 →** [下一個檔名](./檔名.md)`
- README 的導覽表用相對連結指向 01/02/03

## 程式碼區塊內的中文註解

程式碼區塊(```)裡的中文註解**維持半形標點**,因為它在 monospace 區塊裡視覺上屬於「程式碼」情境,全形標點會顯得突兀。

## 可選:大量文字的批次轉換工具

若一次要處理很多段中文正文,可用這個 Python 做法批次把半形標點轉全形——**只在 ``` 區塊之外、且符號緊鄰中文字時**才轉,避免動到程式碼:

```python
import re, glob

CJK = r'一-鿿㐀-䶿＀-￯　-〿「」『』、，。：；？！（）'
pairs = [(',', '，'), (';', '；'), (':', '：'), ('?', '？')]

def convert(line):
    for half, full in pairs:
        h = re.escape(half)
        line = re.sub(rf'(?<=[{CJK}]){h}', full, line)   # 前面是中文
        line = re.sub(rf'{h}(?=[{CJK}])', full, line)     # 後面是中文
    return line

for fn in glob.glob('複習筆記/*.md'):
    out, in_fence = [], False
    for ln in open(fn, encoding='utf-8'):
        raw = ln.rstrip('\n')
        if raw.lstrip().startswith('```'):
            in_fence = not in_fence
            out.append(raw); continue
        out.append(raw if in_fence else convert(raw))
    open(fn, 'w', encoding='utf-8').write('\n'.join(out) + '\n')
```

注意:此工具靠「符號緊鄰中文字」判斷,會漏掉**夾在行內 `code` 之間**的正文標點(兩側是反引號),那幾處需手動補全形。

## ASCII 流程圖:用完整框線(四邊框)

流程圖的方框**用完整四邊框**(`┌─┐ │ … │ └─┘`)。四邊圈起來與其他區塊的區隔明顯、視覺清楚,這是偏好的樣式。

**已知取捨(可接受)**:框內若有中文,中文是全形(佔 2 欄)、框線與空格是半形,右框線可能**不會完全對齊**。這沒關係——看得懂即可,不必為了對齊而拆掉方框。

範例(完整框):

```
Client 發出 HTTP 請求
   │
   ▼
┌─────────────────────────────┐
│  middleware 管線(由上往下)  │
│  1. cors()          ← 跨域   │
│  2. express.json()  ← 解析   │
└─────────────────────────────┘
   │
   ▼
比對路由 → handler
```

想讓右框盡量齊(非強制)可用的小技巧:
- `← 說明` 的箭頭左邊全用半形(程式碼/英文),中文放箭頭右邊。
- 同一框內每行的中文字數盡量相近,右框線的落差就小。
- 真的很在意對齊、框內又有大量中文時,才考慮改用「只有左框線」(`┌─ … │ … └─`)或純箭頭直向流。

## HackMD 專用規則

當產出要貼進 HackMD 時,額外遵守:

- **目錄用內建 `[TOC]`**。它會自動生成可點擊的目錄,且**只讀 H1／H2／H3**。想讓目錄精簡,就把「不需進目錄的深層小標」**降成粗體文字**(`**小標**`),這樣 `[TOC]` 只會列到你要的層級。
- **禁用自訂錨點**:不要用 `# 標題 {#id}`,也不要用 `<a id="x"></a>`。HackMD 的內容過濾會把這些 `id`/`name` 清掉,錨點**一定失效**(`{#id}` 還會露出成文字)。內部導覽一律靠原生 `[TOC]` 或左側目錄。
- **frontmatter**:用 `---` 包住 `title` 與 `tags`。`title` **不含「複習筆記」**——若有書本目錄,那本書的標題已經寫了「複習筆記」,單篇只寫「Week N · 作業主題」,避免重複。
- **單檔不需跨檔連結**:不要放 `[← 回索引]`、`**下一份 →**` 這類相對連結(那是四檔資料夾格式用的;貼到 HackMD 會失效)。
- **引用區塊多行**同樣要在中間放空的 `>`(見上文),HackMD 也吃這套。
