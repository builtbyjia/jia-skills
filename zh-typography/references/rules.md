## 一、標點全形／半形

判斷主體：**這個標點在替哪一個句子斷句？** 替中文句子斷句 → 全形；替英文句子或程式碼斷句 → 半形。與標點左右緊鄰的是不是英文**無關**。

| 標點 | 中文句子（全形） | 英文／程式碼（半形） |
|---|---|---|
| 逗號 | `，` | `,` |
| 句號 | `。` | `.` |
| 冒號 | `：` | `:` |
| 分號 | `；` | `;` |
| 問號 | `？` | `?` |
| 驚嘆號 | `！` | `!` |
| 頓號（僅中文有） | `、` | —— |

頓號 `、` 只用於中文詞語並列（`前端、後端、資料庫`），英文並列用逗號加空格或斜線。

## 二、盤古之白（中英數之間的空格）

在**中文字**與**英文字母／數字**相鄰處，加一個**半形空格**，讓兩種文字不擠在一起，閱讀更順。名稱來自「盤古開天，劈開全形與半形之間的混沌」。

要加空格：
- 中↔英：`重開 Claude Code 就生效`、`一個 skill 就是一個資料夾`
- 中↔數：`收錄 3 個 skill`、`第 2 版`

不加空格（空格是「字與字之間」的事，遇到標點就讓標點負責間距）：
- 英文／數字**緊貼全形標點**時不加：`（四檔資料夾 or HackMD 單檔）` 的 `）` 前不加空格；`目錄即可：` 的 `：` 前不加空格。
- 行內程式碼、URL 內部不受此規則影響。

注意：盤古之白是**空格**，不是標點，跟全形／半形的判斷是兩件獨立的事，兩者都要顧。

## 三、引號

| 語境 | 用法 |
|---|---|
| 中文句子中的引用、強調 | 全形 `「」`，內層再引用用 `『』` |
| 英文句子中的引用 | 半形 `"..."` 或 `'...'` |
| 中文句子中提到英文詞並想加引號 | 用中文引號 `「」`，例 `他說的「skill」是指…` |

全篇統一，不要 `「」` 和 `""` 混用。

## 四、破折號、連接號、刪節號

| 符號 | 正確寫法 | 常見錯誤 |
|---|---|---|
| 破折號（轉折、補述） | 全形 `——`（兩個佔四格，視覺上一長橫） | `--`、單個 `—`、`~` |
| 刪節號（省略、語氣未盡） | 全形 `……`（兩個佔六點） | `...`、`。。。` |
| 連接號（範圍、區間） | `～` 或 `–`（如 `9～12 月`、`頁 10–20`） | 中文範圍用 `-` 半形連字號 |

一致性最重要：全篇破折號要嘛都 `——`、要嘛都不用，別混。

## 五、括號

中文句子裡的括號用**全形 `（）`**，**即使括號內容是英文或程式碼名詞**——因為括號是這個中文句子的結構。

```
✓ 這是純 markdown（study-notes 另含 references），不含腳本
✗ 這是純 markdown(study-notes 另含 references),不含腳本
```

例外：括號**整組**屬於程式碼或英文語境（例如寫在行內 `code` 或程式碼區塊裡的 `foo(bar)`），用半形。

## 六、不可碰的聖域

以下情境**內部標點一律不動**，改了會弄壞語意或功能：

1. **程式碼區塊**（``` 或縮排圍住的）：整段是程式碼，逗號、冒號、括號都半形，且不加盤古之白。
2. **行內 `code`**：反引號內的內容原封不動，`req.params.id`、`{ id: 1 }` 都不改。
3. **URL 與路徑**：`https://a.com/b?x=1`、`~/.claude/skills/` 內的符號不動。
4. **檔名與副檔名**：`README.md`、`.gitignore` 不動。
5. **連接／並列符號**：`/`（`前端/後端`）、`+`（`A + B`）、`→`（`Fork → PR`）維持半形；它們是符號不是句讀。

注意常見陷阱：**行內 `code` 外面**的正文標點仍要依主體語言判斷。`合併 `.md`，自帶 …` 這個逗號在 code 外、替中文句子斷句 → 全形。

## 七、疑難個案

**Q：整句都是英文，但夾一兩個中文詞？**
主體是英文 → 標點半形。例：`Use the study-notes skill, then commit.`（逗號半形）。這種情況在中文文件裡少見，多半是引用整句英文。

**Q：條列項的標籤冒號？**
`- 觸發：講「作業寫完了」…` 標籤是中文 → 全形 `：`。

**Q：數字與單位、時間？**
`3 個`、`5 分鐘` 加盤古之白；`3:30`（時間）在中文句子裡可寫 `3：30` 或維持 `3:30`，但時間慣例上半形冒號可接受，全篇一致即可。

**Q：Emoji 前後？**
Emoji 視同一個字元，前後照中英間距習慣留半形空格即可，不影響標點判斷。

**Q：Markdown 語法符號（`#`、`-`、`|`、`*`）？**
那是 markdown 語法不是內文標點，一律半形、不動。
