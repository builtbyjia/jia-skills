# 繁體中文排版校對（zh-typography）

把一段繁體中文文字的標點與排版修到乾淨、一致、專業。核心不是死記符號表，而是掌握**一條判斷原則**，其餘都是它的延伸。

## 核心原則：主體語言決定標點

**標點屬於它所在的那個句子，不屬於旁邊那個字。** 先看這個句子的主體語言是中文還是英文，再決定標點全形或半形：

- 句子主體是**中文** → 句讀（逗號、句號、冒號、分號、問號、驚嘆號、括號、引號）**一律全形**，**即使標點緊鄰英文詞、數字或行內程式碼**。
- 句子主體是**英文**，或身處**純程式碼／URL** 情境 → 用半形。

為什麼這樣分？因為標點的作用是切分「這個句子」的語氣與結構。中文句子裡出現一個 `Claude` 或 `.md`，那只是句子裡的一個詞，逗號仍然是在替這個中文句子斷句，所以跟著中文走、用全形。反過來，把中文句子的逗號寫成半形，會讓字距忽寬忽窄、閱讀時頓挫，也顯得沒校對過。

這條原則常見的**錯誤直覺**是「符號緊鄰英文就用半形」——那是錯的，別用鄰近字元判斷，要用主體語言判斷。

## 快速判斷流程

遇到一個標點，依序問：

1. 它在**程式碼區塊、URL、或行內 `code` 的內容裡**嗎？→ 原封不動，半形，不要碰。
2. 它所在的**句子主體是中文**嗎？→ 是就用全形，哪怕左右鄰接英文詞或數字。
3. 它是**盤古之白**（中英數之間的空格）嗎？→ 那是空格不是標點，該補就補（見下）。
4. 它是**斜線 `/`、加號 `+`、箭頭 `→` 這類連接／並列符號**嗎？→ 維持半形。

## 規則速查

完整規則與更多對照範例在 `references/rules.md`，這裡先給最常用的：

| 情境 | 用法 | 範例 |
|---|---|---|
| 中文句子的逗號、句號、冒號、分號、問號、驚嘆號 | 全形 `，。：；？！` | `想確認，輸入 /`、`目錄即可：` |
| 中文句子的括號、引號 | 全形 `（）「」`，即使括號內是英文 | `（四檔資料夾 or HackMD 單檔）` |
| 標點緊鄰英文詞／數字，但句子是中文 | **仍全形** | `對應情境，Claude 就會`、`合併 .md，自帶` |
| 中英文、中文與數字之間 | 補**半形空格**（盤古之白） | `重開 Claude Code 就生效`、`收錄 3 個` |
| 英數字連寫的完整 token（產品名、版本號） | **不拆、不加空格** | `Vue3`、`Web3`、`S3`、`ES2020`、`H2O` 保持原樣 |
| 破折號（中文句子表轉折／補述） | 全形 `——`（佔兩格，連用） | `見諒——在 PR 裡說明` |
| 刪節號 | 全形 `……`（六點，連用） | `等等……` |
| 程式碼區塊、URL、行內 `code` 內容 | 一律半形，不動 | `git clone https://...`、行內碼 `.md` 本身 |
| 並列斜線、連接、箭頭 | 半形 | `課程/語言/框架`、`README + 索引`、`Fork → PR` |

## 常見錯誤對照

這些都取自真實案例——句子是中文，標點卻被寫成半形，或誤以為鄰接英文就該半形。`✗` 是常見錯誤，`✓` 是修正後（用等寬字看，全形與半形的字距差別會很明顯）：

```
範例 1（鄰接英文詞）
✗ 一些好用的 skill,整理出來開源分享,希望對你有幫助
✓ 一些好用的 skill，整理出來開源分享，希望對你有幫助

範例 2（逗號後接英文）
✗ 只要講到對應情境,Claude 就會自動照著做
✓ 只要講到對應情境，Claude 就會自動照著做

範例 3（緊鄰行內程式碼，反引號內容不動、外面的逗號要全形）
✗ 一份合併的 `.md`,自帶 `[TOC]` 目錄
✓ 一份合併的 `.md`，自帶 `[TOC]` 目錄

範例 4（括號內含英文，括號仍用全形）
✗ 這是純 markdown(study-notes 另含 references),不含腳本
✓ 這是純 markdown（study-notes 另含 references），不含腳本

範例 5（冒號、問號）
✗ 複製到你的個人 skills 目錄即可: 想要其中一個?複製那份就好
✓ 複製到你的個人 skills 目錄即可：想要其中一個？複製那份就好
```

## 校對一份既有文件的流程

1. **通讀一遍**，先辨識哪些區塊是「不可碰」的：程式碼區塊（``` 圍住的）、URL、行內 `code` 的內容。這些內部標點一律不動。
2. **逐處判斷**其餘的中文正文：套用上面的快速判斷流程，把中文句子裡該全形的句讀改全形。
3. **補盤古之白**：中英文、中文與數字之間補半形空格；但緊貼標點的地方不補（例如 `全形），不含` 不需要在 `）` 前後加空格），且**英數字連寫的完整 token 不要拆開**（`Vue3`、`Web3` 是一個名詞，別改成 `Vue 3`）。盤古之白只作用在「中文 ↔ 英/數」的邊界，不動英數字內部。
4. **檢查一致性**：破折號、刪節號、引號全篇統一（不要一下 `--` 一下 `——`、一下 `"" ` 一下 `「」`）。
5. **可選：用腳本粗轉**大量文字後再人工複查（見下）。

## 輔助腳本（可選，不能全靠它）

`scripts/convert_punctuation.py` 能把「緊鄰中文字的半形句讀」批次轉全形，處理大量文字時省事。**但它只是加速器，不是判斷者**：純規則無法理解「主體語言」，它靠「符號緊鄰中文字」這個近似條件，因此會**漏掉夾在行內 `code` 兩側的正文標點**（兩邊是反引號不是中文字），這些要人工補。跑完務必自己再讀一遍。

用法：`python scripts/convert_punctuation.py <檔案或 glob>`。它會跳過 ``` 程式碼區塊。

## 心法

- **主體語言優先，鄰近字元是陷阱**：中文句子的標點跟中文走，別被句中的英文詞帶偏。
- **校對是修形式，不是改寫**：只動標點與空格，不要順手改掉作者的字句、語氣或刪節號——把 `……` 改成 `，`、把 `Vue3` 改成 `Vue 3` 都是越權。
- **不確定就問「這個標點在替哪個句子斷句」**：替中文句子斷句就全形，替程式碼／英文斷句就半形。
- **程式碼與 URL 是聖域**：括號、冒號、逗號在那裡面都不動，改了會弄壞語法或連結。
- **一致性是專業感的來源**：破折號、引號、刪節號全篇同一種寫法，比個別正確更重要。
